%N Minimum number of hexagonal cells in a connected polyhex that contains every one-sided n-hex as a rotation-and-translation subset.

%S 1,2,4,7,11,15,21

%O 1,1

%C A one-sided n-hex is a connected set of n unit hexagonal cells on the hex lattice, counted up to rotation only; reflections are distinct pieces. The one-sided polyhex count is A006535, with A006535(n) = 1, 1, 3, 10, 33, 147, 620, 2821, 12942, 60639, ... for n = 1, 2, 3, ... .

a(n) is the minimum number of cells in a connected polyhex that contains every one-sided n-hex under some rotation and translation of the piece.

Each reported value a(n) for n = 1..7 is proved by SAT search inside an (n+1) X (n+1) axial rectangle with a machine-verified UNSAT proof at size a(n) - 1 (drat-trim "s DERIVATION" for n = 1..6; cadical --lrat + lrat-check "c VERIFIED" for n = 7, where the 1.1 GB binary DRAT exceeded drat-trim's practical memory budget). Every value is also cross-checked by an independent pure-Python geometric containment verifier with a disjoint code path from the solver. Both stacks agree on n = 1..7.

No closed-form expression matches a(1..7). The observed growth is polynomial of order 2 with n^2 / 3 <= a(n) <= 5 n^2 / 8 empirically for n = 1..7 (UNVERIFIED as a general bound).

%H _Peter Exley_, <a href="https://github.com/pexley-math/oeis-A395434">Solver code, verifiers, DRAT/LRAT certificates, figures, and paper</a>, GitHub.

%e For n = 1, a(1) = 1: a single hexagonal cell contains the unique one-sided 1-hex.
%e For n = 2, a(2) = 2: a 2-cell domino contains the unique one-sided 2-hex.
%e For n = 3, a(3) = 4: a 4-cell connected polyhex with bounding box 2 X 3 contains all 3 one-sided 3-hexes.
%e For n = 7, a(7) = 21: a 21-cell connected polyhex with bounding box 8 X 5 contains all 620 one-sided 7-hexes under rotation + translation.
